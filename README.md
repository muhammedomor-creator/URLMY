# 🔗 প্রিমিয়াম ইউআরএল শর্টনার ও ড্যাশবোর্ড (Premium URL Shortener)
একটি অত্যন্ত আধুনিক, নিরাপদ এবং প্রফেশনাল সিঙ্গেল পেজ অ্যাপ্লিকেশন (SPA) ইউআরএল শর্টনার। এটি মূলত বাংলাদেশী ব্যবহারকারীদের লক্ষ্য করে তৈরি করা হয়েছে, যেখানে রয়েছে হোয়াটসঅ্যাপ ওটিপি (WhatsApp OTP) ভিত্তিক রেজিস্ট্রেশন, পাসওয়ার্ড রিকভারি, ডাইরেক্ট রিডাইরেকশন এবং ভুল করে ডিলিট হওয়া রোধে ১৫ সেকেন্ডের অনন্য আনডু (Undo) ডিলিট সিস্টেম।
🌐 **লাইভ ওয়েবসাইট লিংক:** https://urlmy.vercel.app

## ✨ আকর্ষণীয় ফিচারসমূহ (Core Features)
 * **⚡ ডাইরেক্ট রিডাইরেকশন (No Intermediate Screen):** আমাদের কাস্টম শর্ট লিংকে ক্লিক করা মাত্রই কোনো প্রকার মধ্যবর্তী বিজ্ঞাপন বা লোডিং স্ক্রিন ছাড়াই সরাসরি মূল ওয়েবসাইটে রিডাইরেক্ট করা হয়।
 * **💬 হোয়াটসঅ্যাপ ওটিপি সিস্টেম (WhatsApp OTP System):** ওটিপি এপিআই https://otp-api-hmrz.onrender.com/send-otp ব্যবহারের মাধ্যমে সরাসরি ব্যবহারকারীর হোয়াটসঅ্যাপে ৬ ডিজিটের ওটিপি কোড পাঠানো হয়।
 * **⏳ ১৫ সেকেন্ডের আনডু ডিলিট (15s Undo Countdown):** যেকোনো লিংক ডিলিট করতে চাইলে সেটি সাথে সাথে মুছে যায় না। স্ক্রিনে ১৫ সেকেন্ডের টাইমার কাউন্টডাউন শুরু হয়। ব্যবহারকারী চাইলে **"Undo / বাতিল করুন"** বাটনে ক্লিক করে ডিলিট হওয়া রোধ করতে পারেন।
 * **📌 মনে রাখার নোট/ট্যাগ (Custom Notes):** আপনি কোন উদ্দেশ্যে লিংকটি ছোট করেছেন তা লিখে রাখার জন্য বিশেষ কাস্টম নোট বা ট্যাগ যুক্ত করার চমৎকার সুবিধা রয়েছে।
 * **🛡️ নিরাপদ লোকাল সেশন (Local Session Management):** একবার লগইন করলে ব্রাউজার সেশন মনে রাখে, ফলে প্রতিবার নতুন করে লগইন করার ঝামেলা নেই।

## 🔐 ফায়ারবেস সিকিউরিটি রুলস (Firebase Security Rules)
সঠিকভাবে ডাটা রাইট ও রিড নিশ্চিত করার জন্য ফায়ারবেস ফায়ারস্টোর ডাটাবেজের **Rules** ট্যাবে নিচের কোডটি সেট করুন:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

```
## 🏢 এন্টারপ্রাইজ বা বাণিজ্যিক ডাটাবেজ পরিবর্তন নির্দেশিকা (Enterprise Database Migration Guide)
বর্তমানে প্রজেক্টটি ব্যক্তিগত বা সীমিত পরিসরে ব্যবহারের জন্য ডিজাইন করা হয়েছে। যদি আপনি প্রজেক্টটিকে বড় কোনো প্রতিষ্ঠান বা মিলিয়ন ব্যবহারকারীর জন্য বাণিজ্যিক আকারে (Enterprise Level) ব্যবহার করতে চান, তবে সিকিউরিটি এবং পারফরম্যান্স বাড়াতে নিচের পরিবর্তনগুলো করতে হবে:
### ১. ফায়ারবেস সিকিউরিটি লক করা (Secure Rules)
বর্তমানে রুলস ওপেন (allow read, write: if true) করা আছে, যা বাণিজ্যিক ব্যবহারের জন্য অনিরাপদ। এন্টারপ্রাইজ লেভেলে ফায়ারবেস সিকিউরিটি রুলসটি পরিবর্তন করে শুধুমাত্র অথেনটিকেটেড ইউজারদের এক্সেস দিতে হবে:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // শুধুমাত্র লিঙ্ক ও ইউজার ডকুমেন্ট সিকিউর করা
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    match /links/{linkId} {
      allow read: if true; // রিডাইরেক্টের জন্য লিংক রিড সবার জন্য উন্মুক্ত থাকবে
      allow write: if request.auth != null; // শুধুমাত্র রেজিস্টার্ড ইউজার লিখতে পারবেন
    }
  }
}

```
### ২. নিজস্ব ডোমেন ও হোয়াটসঅ্যাপ ওটিপি গেটওয়ে
বাণিজ্যিক উদ্দেশ্যে Render-এর ফ্রি সার্ভার (Api) ব্যবহার না করে একটি প্রিমিয়াম বা ডেডিকেটেড হোয়াটসঅ্যাপ এপিআই সার্ভিস ব্যবহার করা উচিত (যেমন: Twilio, Baileys/MD self-hosted VPS)। এতে ওটিপি ডেলিভারি স্পিড অনেক গুণ বৃদ্ধি পাবে এবং কোনো ডাউনটাইম থাকবে না।
### ৩. ইনডেক্সিং ও পারফরম্যান্স কুয়েরি (Firestore Indexing)
হাজার হাজার ব্যবহারকারী যখন লাখ লাখ লিংক তৈরি করবেন, তখন ডাটাবেজে দ্রুত কুয়েরি করার জন্য ফায়ারস্টোরে userId এবং createdAt ফিল্ডের উপর **Composite Indexes** তৈরি করে নিতে হবে। এতে ড্যাশবোর্ডের লোডিং স্পিড অনেক গুণ বৃদ্ধি পাবে।
